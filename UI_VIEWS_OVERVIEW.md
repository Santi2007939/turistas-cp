## Color Coding (Tailwind CSS)

### Role Badges
- **Admin**: Purple background (`bg-purple-100`), Purple text (`text-purple-800`)
- **Student**: Blue background (`bg-blue-100`), Blue text (`text-blue-800`)

### Status Badges
- **Active**: Green background (`bg-green-100`), Green text (`text-green-800`)
- **Inactive**: Red background (`bg-red-100`), Red text (`text-red-800`)

### Current Member Badges
- **Yes**: Green background (`bg-green-100`), Green text (`text-green-800`)
- **No**: Gray background (`bg-gray-100`), Gray text (`text-gray-800`)

### Action Buttons
- **Approve**: Green text (`text-green-600`), hover green (`hover:text-green-900`)
- **Deactivate**: Red text (`text-red-600`), hover red (`hover:text-red-900`)
- **Mark Member**: Blue text (`text-blue-600`), hover blue (`hover:text-blue-900`)
- **Remove Member**: Gray text (`text-gray-600`), hover gray (`hover:text-gray-900`)

## Key UI Components

### 1. Login Component
- Email and password inputs
- Error message display (red background)
- Success message display (green background)
- Role-based redirect logic

### 2. Register Component
- Form fields (username, email, password, full name, codeforces handle)
- Different messaging for first user vs regular users
- Redirect logic based on role

### 3. Admin Dashboard Component
- Navigation bar with admin indicator
- User management table with:
  - User information columns
  - Status indicators (colored badges)
  - Action buttons (contextual based on user state)
- Success/error message display
- Real-time table updates

### 4. Student Dashboard Component
- Navigation bar with student features
- Feature cards for different sections
- Standard dashboard view

## Accessibility Features

- Clear visual distinction between roles (color + text)
- Action buttons with descriptive titles
- Success/error messages clearly visible
- Logical tab order for keyboard navigation
- Screen reader friendly badges and labels

All views are responsive and work on:
- Desktop (>1024px)
- Tablet (768px - 1023px)
- Mobile (< 768px)

Tables on mobile may scroll horizontally or stack vertically depending on implementation.
